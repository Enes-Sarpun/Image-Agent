# Image Agent — Geliştirme Devamı

## Projenin Özeti

"Image Agent" — Gemini API kullanarak bir görselin AI tarafından üretilmiş mi yoksa gerçek fotoğraf mı olduğunu tespit eden Python tabanlı bir AI agent. Lokalde çalışan CLI uygulaması, GitHub'da açık kaynak olarak yayında.

Kullanıcı bir görsel yüklüyor, agent çok katmanlı bir analiz yapıyor: önce hızlı kontroller (cache, watermark tespiti, EXIF), sonra forensic önişleme (ELA + FFT + edge analizi), sonra LLM vision (Gemini 2.5 Flash). Düşük confidence durumlarında multi-pass reasoning ve Google Search grounding fallback'leri var.

Hedef: Portfolio'da gösterilebilecek + GitHub'da kullanıcılara ulaşacak kalitede, doğru cevap veren ve iyi UX sunan bir tool.

## Çözülmüş Problemler

1. Eski/düşük kalite AI tespiti: %95+ doğrulukla çalışıyor. Gemini watermark'lı görseller, klasik Stable Diffusion/Midjourney v3-4 üretimleri kolayca yakalanıyor.

2. Forensic edge analizi: ELA + FFT + Edge pattern detection ekledik. AI üretimi görsellerin "uniform tekstür", "tekrarlanan pattern", "köpük dokusu" gibi imzalarını sayısal olarak yakalıyor (uniformity_std, texture_repetition, density_entropy metrikleri).

3. LLM Real bias kırma: Forensic skoru LLM'e context olarak veriyoruz. LLM artık "kör" karar vermek yerine forensic ipuçlarını da değerlendiriyor.

4. Cache sistemi: SQLite + perceptual hash. Aynı görsel ikinci kez analiz edilmiyor, ~100ms'de cache'den dönüyor.

5. Watermark tespiti: Gemini sparkle watermark'ı sağ alt köşede pixel analiziyle yakalanıyor.

## Çözülmemiş Problemler

1. Yeni nesil fotogerçekçi AI tespiti: Gemini Imagen 3, Flux, Midjourney v6 gibi yeni modellerin watermark'sız üretimleri hâlâ kaçırılıyor. Test setinden örnek: "Zürafa" görseli (çöl manzarasında zürafa, Gemini üretimi) "real" olarak %95 confidence ile tahmin ediliyor. Modern AI üretimleri çok temiz olduğu için forensic sinyaller zayıf, LLM "real" demeye yatkın.

2. Confidence kalibrasyonu eksik: LLM ham confidence skorunu olduğu gibi kullanıyoruz. "%98 real" diyor ama yanılmış oluyor. Bunu otomatik kalibre eden bir mekanizma yok henüz.

3. Sadece CLI: Şu an terminal'den çalışıyor, görsel arayüz yok. Portfolio için ve potansiyel kullanıcılar için arayüz şart.

4. Batch yok: Tek tek görsel analizi var. Klasör veya çoklu görsel yok.

5. Profesyonel sunum eksik: README minimal, ekran görüntüsü yok, demo gif yok, mimari diyagramı yok.

### Analiz Pipeline

Image input
  ↓
[1] Perceptual hash → Cache check
  ├─ Hit  → Return (~100ms)
  └─ Miss ↓
[2] EXIF + Watermark
  ├─ Watermark found (>95%) → Return (~300ms)
  └─ Not found ↓
[3] Forensic preprocessing (ELA + FFT + Edge)
  ↓
[4] LLM Vision (Gemini Flash, forensic context'li)
  ├─ High confidence (≥75%) → Return (~5-10s)
  └─ Low confidence ↓
[5] Multi-Pass Reasoning (3 pass)
  ├─ Pass 1: AI evidence search
  ├─ Pass 2: Real evidence search
  ├─ Pass 3: Synthesis
  ├─ High confidence → Return (~15-20s)
  └─ Still low ↓
[6] LLM + Google Search fallback (~30s)

## Bağlam: Önemli Kararlar

- Model: Gemini 2.5 Flash kullanıyoruz. 2.5 Pro denedik ama yavaşlık ve maliyet açısından Flash'ta kaldık.
- Free tier: Geliştirme tamamen ücretsiz tier'da yapılıyor (10 RPM, 250 RPD).
- Senkron: Async kullanmıyoruz, basit ve debug edilebilir kalsın diye.
- Lokalde çalışma: Hosting yok, kullanıcı kendi makinasında çalıştıracak.
- Hibrit ML modeli yok: Eski projemde EfficientNetV2-S + FFT CNN hibrit modeli vardı. Onu Image Agent'a dahil etmedik, sadece LLM odaklı gittik.

## Geliştirme Planı

### Faz 1: Multi-Pass Reasoning Test ve İnce Ayar
Durum: Multi-pass kodu yazıldı (multi_pass_analyzer.py). Test edilmedi henüz.

Yapılacaklar:
- Multi-pass'i test setiyle çalıştır (özellikle Zürafa gibi yanılgı durumlarında)
- Promptlarda ince ayar: AI evidence prompt'u daha agresif yap, real evidence prompt'u daha şüpheci yap
- Eğer hâlâ "real" diyorsa, prompt'a "modern fotogerçekçi AI" örnekleri ekle (few-shot)
- Synthesis pass'inin confidence kalibrasyonunu sıkılaştır

Başarı kriteri: Test setinde en az %80 doğruluk + watermark'sız Gemini görsellerinin %50+'sını yakalayabilme.

### Faz 2: Confidence Calibration Layer
Durum: Yapılmadı.

Yapılacaklar:
- src/tools/confidence_calibrator.py dosyası oluştur
- Şu kuralları implement et:
  - Watermark yokken raw_confidence > 90 ise → max 80'e indir
  - Forensic AI score > 50 ama LLM "real" diyorsa → confidence -%20
  - EXIF camera info varsa "real" yönüne ek %10
  - EXIF tamamen boşsa "ai" yönüne ek %5
  - Birden fazla kaynak (LLM + forensic + EXIF) aynı yöne işaret ediyorsa confidence boost
- Agent'a entegre et, final result'tan önce çağır

Başarı kriteri: "Yanlışken yüksek confidence" sorunu kalmamalı. Yanlış cevaplar artık düşük confidence ile geleceği için kullanıcı uyarılmış olacak.
